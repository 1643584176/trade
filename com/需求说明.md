代码问题:
    1.禁止使用本地时间,只能用xauusd时间
    2.禁止使用任何模拟数据
    3.禁止重复开仓
    4.真实交易必须使用MagicNumberEnum文件下的号码
    5.模型训练只能是单独的文件,需要使用真实的mt5数据
    6.回测文件只能加载模型禁止训练模型
    7.真实交易只能加载模型,并且还能增量训练
    8.所有账户只能操作自己的margicNumber号
    9.手数都是需要可配置
    10.需要区分当前时间是否处于交易时间,不是交易时间就等待1小时
交易问题:
    1.满足ftmo挑战规则,每日交易最大亏损值不能大于4.5%,超过需要立即平仓,需要考虑启动项目是今天已经做过交易了,所以是当日持仓的最大盈亏和当日已交易的亏损
    2.第一次挑战如果是10万,总盈利到达110030立即平仓,总账户余额不能小于99020,如果是1万账户,总盈利到达11030立即平仓,总账户余额不能小于99020 
    3.0.2手止损120美金(波动600点位),举例就是1994到2000!,止盈是200美金(波动1000点位)
    4.整体信号有上涨\观望\下跌三个重点信号
    5.如果综合上涨概率 > 0.7 → 上涨信号;如果综合上涨概率 < 0.3 → 下跌信号 ;如果0.3 ≤ 综合上涨概率 ≤ 0.7 → 观望信号
模型问题:
    1.只做日内短线交易
    2.选用的模型必须有自我进化能力
    3.至少需要训练3个模型 一个是m1周期,一个是m5周期,一个是m15周期,m1的数据只有有60天,m5的数据可以有一年,m15数据可以用一年
    4.模型预测m1使用30-50根预测1-3根m1,m5使用120根预测1-3根m5,m15使用200根预测1-4根m15k线
    5.M5权重最高（0.5-0.6）：主要决策依据，数据最丰富 ,M15权重中等（0.3-0.4）：趋势确认,M1权重较低（0.1-0.2）：时机优化和风险预警
    6.m1提供超短期市场情绪和快速变化,快速检测异常波动，提供早期风险信号; M5作为主要交易周期;M15提供趋势确认,识别当前处于趋势、震荡还是反转状态等趋势
    7.确保止盈止损设置正确,比如买空的止盈止损和买多的止盈止损,根据波动幅度设置止盈止损,不支持动态调整止盈止损
    8.信号反向就平仓;交易信号为观望且当前持仓订单盈利超过90美元时平仓;每日收盘前必须平仓;
    9.交易时段:M1 模型：60 天数据，市场微观结构变化快 → 每 4 小时更新一次（覆盖亚盘 / 欧盘 / 美盘不同时段）； M5 模型：1 年数据，中周期规律稳定 → 每日收盘后更新一次； M15 模型：1 年数据，趋势规律最稳定 → 每周更新一次。
特征问题:
    1.选用XGBoost/LightGBM来训练模型
